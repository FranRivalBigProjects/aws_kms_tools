// KMS pricing model

How AWS KMS Key Cost Is Calculated

AWS KMS charges have two components: key storage and API requests. Each Customer Managed Key (CMK) costs $1.00/month regardless of usage. API requests are charged per 10,000 operations, with the rate depending on key type — symmetric operations are cheapest, asymmetric operations cost over 3x more per request.

The first 20,000 symmetric key API requests per month are free. This free tier resets monthly and covers the vast majority of low-volume use cases. For high-throughput encryption workloads (millions of requests/month), the API request cost typically exceeds the key storage cost.

Key rotation (automatic annual rotation for symmetric keys) is included in the $1/month key price — there is no additional charge for enabling automatic rotation.

// Optimization

How to Reduce AWS KMS Costs

Use data key caching to reduce API requests
The AWS Encryption SDK supports data key caching — reusing a GenerateDataKey response to encrypt multiple messages before calling KMS again. Instead of one KMS call per object, you can encrypt thousands of objects with a single cached data key. This can reduce KMS API costs by 99% for high-volume encryption workloads like S3 object encryption or DynamoDB item encryption.

Audit and delete unused KMS keys
Every CMK costs $1/month regardless of whether it is used. Audit your KMS key list with CloudTrail: if a key shows zero usage in the last 90 days, schedule it for deletion (minimum 7-day waiting period). KMS has a GetKeyRotationStatus and CloudTrail has kms.amazonaws.com events you can query. Unused keys from old projects are a common source of silent KMS spend.

Use AWS Managed Keys instead of CMKs for non-regulated data
If you do not need custom key policies, cross-account access, or manual key rotation control, AWS Managed Keys are free. S3 SSE-S3 and SSE-KMS with AWS managed keys incur no key storage cost. Switch to CMKs only when compliance requirements explicitly require customer-controlled keys or when you need usage audit logging at the key level.

Prefer symmetric keys over asymmetric keys for bulk encryption
Asymmetric key API requests cost $0.10/10K vs $0.03/10K for symmetric — more than 3x more expensive. If your use case is data encryption (not digital signatures or public key operations), symmetric keys are the correct and cheaper choice. Asymmetric keys are required for RSA signing, ECDSA, and scenarios where the public key must be shared externally.

Symmetric key requests: $0.03 per 10,000 (first 20,000 free/month). Asymmetric key requests: $0.10 per 10,000. HMAC key requests: $0.15 per 10,000. Prices are per the AWS US East region and vary slightly in other regions. Requests include Encrypt, Decrypt, GenerateDataKey, Sign, Verify, GenerateMac, and other cryptographic operations.
A typical application with 5 CMKs and 1 million symmetric requests/month: Keys = 5 × $1 = $5/month. Requests = (1,000,000 - 20,000 free) / 10,000 × $0.03 = $2.94/month. Total = ~$8/month. High-volume applications encrypting 100M objects/month: $5 + (100M / 10K × $0.03) = $5 + $300 = $305/month — data key caching could reduce the $300 request cost to under $1.
